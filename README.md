# 随手记

好记性不如烂笔头。

这本 GitBook 是我的随身笔记本：灵感、折腾、排障、脚本、部署片段，都先记下来。很多想法不是因为宏大才值得记录，而是因为它们稍纵即逝；不写下来，第二天早上可能就永远消失了。

Richard Branson 说，笔记本是他最重要的商业工具之一。真正有价值的不是“记了多少”，而是能不能把一闪而过的念头变成明确目标，再一步步落成行动。

所以这里不追求漂亮完整，也不假装系统化。先记，后整理；先留下线索，再把能做的东西做出来。

## 这里会放什么

- 临时想到但不想丢的想法
- Mac、服务器、Docker、Surge、Hermes 的配置记录
- 可复用的脚本和排障方法
- 视频、媒体、自动化相关的操作片段
- 那些当时看起来很小、后来可能救命的细节

## 使用原则

- 能短就短，先保留现场。
- 能验证就验证，不只写感觉。
- 能行动就拆成下一步，不让想法停在纸面上。
- 敏感信息不入库，只写方法和占位符。

笔记不是为了收藏念头，是为了给未来的自己留一条能回来的路。
